Webbutveckling 1

Begreppsförklaring

Det är grundstrukturen för alla webbsidor du ser på nätet. Alltså layouten såsom styckindelning och rubriker.

Det är det som gör att just din sidan ser annorlunda ut. Alltså typsnitt, storlek och hemsidans färger.

Element:

En del av ett Htmldokument, som börjar på en startag och slutar på en sluttag. Exempelvis h1,img,p.

ID:

Specifierar ett unikt id för ett element och kan bara ha ett per element. Detta ID kan inte användas på flera element. Det måste också:

    Innehålla en bokstav
    Inga mellanslag

Class:

Klass atributet specifierar en eller flera "klassnamn" för ett element. Det används mestadels för att kunna länka med klassen i ett stylesheet.

Meta:

Meta taggen i Html är information om data i dokumentet. Taggen går alltid innanför head-element och brukar användas för att:

    Teckenuppsättning
    Nidbeskrivning
    Nyckelord
    Dokumentförfattare
    Visningsinställningar

Responsive:

Responsive menas med att hemsidan ser bra ut på alla enheter. När den är responsive används bara CSS och Html. Det brukar ofta benämnas som "Responsiv hemsida" och det är alltså när den krymper, växer eller vidgas så att det ser bra ut.

Http:

Http står för "Hyper Text Transfer Protocol" och det handlar om att man ska kunna komma åt hemsidorna på nätet. Alltså förfrågningar och svar. Process:

    1. En webbläsare skickar en http förfrågning på nätet
    2. En webbserver tar emot förfrågan
    3. Servern kör en applikation som jobbar med förfrågningen
    4. Servern retunerar en Http svar i form av output till webbläsarn
    5. Webbläsaren tar emot svaret

Https:

    1. En webbläsare skickar en http förfrågning på nätet
    2. En webbserver tar emot förfrågan
    3. Servern kör en applikation som jobbar med förfrågningen
    4. Servern retunerar en Http svar i form av output till webbläsarn
    5. Webbläsaren tar emot svaret
